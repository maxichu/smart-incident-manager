 ## F1 — Incident Submission

 Users submit IT incident reports by typing text or uploading a PDF.  
 The extracted text is stored in SQLite with an auto-generated ID and timestamp.

 ### Test Case 1 — Manual Text Entry

 **Input:**
 - Type `Server CPU usage reaches 95% on production node at 14:00 SGT.` in the text area.
 - Leave the PDF uploader empty.
 - Click Submit.

 **Expected Result:**
 - A green success banner: `Incident #1 saved. Text length: 58 characters.`
 - The row exists in the `incidents` table with the submitted text.

 ### Test Case 2 — PDF Upload

 **Input:**
 - Leave the text area empty.
 - Upload a valid text-based PDF (e.g. a one-page report with extractable text).
 - Click Submit.

 **Expected Result:**
 - A green success banner with the incident ID and character count.
 - The extracted PDF text matches the visible content of the PDF.

 ### Test Case 3 — Empty Submission

 **Input:**
 - Leave the text area empty.
 - Do not upload a PDF.
 - Click Submit.

 **Expected Result:**
 - A yellow warning: `Please enter incident text or upload a PDF before submitting.`
 - No new row is inserted into the `incidents` table.
