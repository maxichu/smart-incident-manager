 ## F8 — Question Answering

 RAG workflow: user question -> FTS5 search -> top incidents -> LLM context
 -> generated answer.  Multi-turn conversation via Streamlit session state.

 ### Test Case 1 — Database Question
 **Q:** What happened to the database service?
 **Expected:** Answer references database incidents from FTS5.

 ### Test Case 2 — Security Question
 **Q:** Which incidents were classified as Security?
 **Expected:** Answer references security incidents.

 ### Test Case 3 — Severity Question
 **Q:** What is the most severe incident?
 **Expected:** Answer uses retrieved incident information.

 ### Test Case 4 — Cause Question
 **Q:** What caused the outage?
 **Expected:** Answer generated from retrieved incidents only.

 ### Test Case 5 — Out of Scope
 **Q:** What happened to the Mars rover?
 **Expected:** "Not enough information available."

 ### Test Case 6 — Follow-Up (Timing)
 **Q1:** What happened to the database service?
 **Q2:** When did it happen?
 **Expected:** Q2 uses conversation context for "it" reference.

 ### Test Case 7 — Follow-Up (Recommendations)
 **Q1:** Which incident was most severe?
 **Q2:** What recommendations were given?
 **Expected:** Follow-up uses conversation context.
