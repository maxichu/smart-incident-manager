 ## F4 — Incident Summarization

 Each incident is automatically summarized using a deterministic Lead-3
 strategy: the first three meaningful sentences are extracted and stored.

 ### Test Case 1 — Five-Sentence Incident

 **Input:** "Server CPU is at 95%. Memory usage is normal. Disk I/O shows
 no issues. Network latency is within expected range. No alerts from
 monitoring system."

 **Expected Result:** Summary = first 3 sentences: "Server CPU is at 95%.
 Memory usage is normal. Disk I/O shows no issues."

 ### Test Case 2 — Two-Sentence Incident

 **Input:** "VPN is down for Singapore users. Network team investigating
 firewall rules."

 **Expected Result:** Summary = both sentences: "VPN is down for Singapore
 users. Network team investigating firewall rules."

 ### Test Case 3 — Single-Sentence Incident

 **Input:** "Database connection timeout on production server."

 **Expected Result:** Summary = that one sentence.
