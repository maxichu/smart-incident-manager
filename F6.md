 ## F6 — Severity Scoring

 Scores each incident by highest-matching severity keyword weight.
 Keywords map to levels: 4=Critical, 3=High, 2=Medium, 1=Low.
 No match defaults to Low.

 ### Test Case 1 — High
 **Input:** Server CPU usage reaches 95%.
 **Expected:** Severity = High (keyword "95%" = weight 3)

 ### Test Case 2 — High
 **Input:** Database service is unavailable.
 **Expected:** Severity = High (keyword "unavailable" = weight 3)

 ### Test Case 3 — Critical
 **Input:** Company-wide outage detected.
 **Expected:** Severity = Critical (keyword "outage" = weight 4)

 ### Test Case 4 — Medium
 **Input:** Application response is slightly slow.
 **Expected:** Severity = Medium (keyword "slow" = weight 2)

 ### Test Case 5 — Low
 **Input:** Informational maintenance notice.
 **Expected:** Severity = Low (keywords "notice" + "informational" = weight 1)
